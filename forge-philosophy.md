# Forge 项目理念

## 什么是 Forge

Forge 是一个基于 Docker 隔离的**自动化代码编译系统**。它允许 AI Agent 在独立容器中完成代码编译工作流，实现端到端的自动化构建。

### 核心能力

| 能力 | 描述 |
|------|------|
| Docker 隔离 | 每个编译会话在独立容器中运行，环境互不干扰 |
| 多构建系统 | 自动识别 cmake/make/autotools 等构建系统 |
| 工件验证 | 确定性验证构建产物（存在性、非空、smoke test） |
| 会话管理 | 完整的状态机和工作流日志，支持重试 |
| 可复现构建 | 自动生成 `build.sh` 脚本用于复现 |

### 工作流程

```
Lead Agent                          Compiler Subagent
    │                                      │
    ├── prepare_compile_session            │
    ├── clone_repository ─────────────────►│
    ├── identify_build_system              │
    │                                     ├── run_container_bash (configure)
    │                                     ├── run_container_bash (build)
    │                                     ├── copy artifacts to /artifacts
    │◄──── submit_build_result ───────────┤
    └── finalize_compile_session          │
```

## 设计理念

### 名称由来

**Forge** (锻造) — 将源代码锻造成可执行产物

这个名字传达了编译系统的本质：
- 像铁匠一样，把粗糙的原材料（源代码）锻造成有用的产品（可执行文件）
- 炉火（Docker 容器）是锻造的场所
- 锤子（Agent）执行精确的敲击动作
- 砧子（验证系统）确保最终产品的质量

### 核心价值

1. **隔离性 (Isolation)**
   - 每个编译会话在完全独立的 Docker 容器中运行
   - 不会污染宿主环境，也不会被其他会话影响

2. **可靠性 (Reliability)**
   - 工件验证机制确保只有通过检验的产物才算成功
   - 确定性构建：相同输入产生相同输出

3. **自动化 (Automation)**
   - 从准备环境到最终验证，全程无需人工干预
   - 适用于 CI/CD、批量构建等场景

4. **可观测性 (Observability)**
   - 完整的操作日志和状态追踪
   - 透明的构建过程

## 视觉设计方向

### 配色方案

| 类型 | 颜色 | 含义 |
|------|------|------|
| 主色 | 深灰/铁灰 `#1a1a1a`, `#2d2d2d` | 金属质感，专业工业化 |
| 强调色 | 橙红 `#ff6b35`, `#f7931e` | 炉火/熔炼，热情而专注 |
| 背景 | 深色 `#0a0a0a` | VS Code 风格，沉浸式 |
| 文字 | 浅灰/白色 | 高对比度，易读 |

### 视觉隐喻

```
火焰 (Flame)        — 熔炉、热情、转化
锤子 (Hammer)       — 锻造、力量、精确
砧子 (Anvil)       — 基础、支撑、坚实
火花 (Spark)        — 能量、创造、瞬间
熔岩灯 (Lava lamp)  — 流动、变化、有机
六边形 (Hexagon)    — 螺丝/螺母、机械、工业
```

### UI 风格参考

- **VS Code** — 专业的深色代码编辑器风格
- **Linear** — 简洁、精确、现代感
- **Railway** — 清晰的构建流程展示
- **Vercel** — 简约工作流

### Logo 设计建议

Forge 的 Logo 可以考虑：

1. **抽象火焰** — 简洁的几何火焰图形，体现"熔炉"概念
2. **锤子+砧子** — 高度抽象的符号组合
3. **六边形+火焰** — 机械感 + 热情的双重暗示
4. **字母 F** — 简约的线条化 F 变形

**重要原则**：
- 简洁易记，在小尺寸下依然清晰
- 避免过于具象（不要直接画鹿）
- 考虑单色版本可用性

### 动效建议

- **熔岩灯效果** — 缓慢流动的渐变，暗示"熔炼"过程
- **火花粒子** — 零星的闪烁，暗示能量释放
- **渐变呼吸** — 柔和的明暗变化，增加生命感
- **Flickering Grid** — 保留但去掉 deer mask，作为技术感的背景

## 产品定位

Forge 是一款面向开发者的**工具型产品**：

- **目标用户**：需要自动化代码编译能力的开发者、DevOps 工程师、AI Agent 开发者
- **使用场景**：代码构建、测试自动化、CI/CD 集成、研究工作流
- **核心诉求**：可靠、自动、可观测

UI 设计应体现：
- **专业感** — 不是消费级产品，而是工程师工具
- **简洁性** — 减少干扰，聚焦核心功能
- **清晰度** — 状态、进度、结果一目了然
